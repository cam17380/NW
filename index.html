ACL Commands (Router / Firewall / L3 Switch)

CommandModeDescription
Standard ACL (1–99) — source IP only
access-list <1-99> permit|deny <network> [wildcard]configAdd standard ACL entry (also used for NAT)
Extended ACL (100–199) — source, destination, protocol, port
access-list <100-199> permit|deny <proto> <src> <srcWC> <dst> <dstWC> [eq <port>]configAdd extended ACL entry
Applying ACLs to interfaces
ip access-group <acl-num> in|outconfig-ifApply ACL inbound or outbound on interface
no ip access-group <acl-num> in|outconfig-ifRemove ACL from interface
Management
no access-list <num>configDelete entire ACL
show access-listsEXECShow all ACLs and applied interfaces

proto: ip (all), icmp, tcp, udp   src/dst: IP + wildcard, any, or host <ip>   eq: port number (tcp/udp only, 1-65535)   Implicit deny all at end of every ACL
Example: access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80  →  Allow HTTP from 192.168.1.0/24 to any destination

Firewall Policy Commands (Firewall only)

CommandModeDescription
firewall policy <seq> permit|deny <src> <srcWC> <dst> <dstWC> <proto> [port]configAdd/update firewall policy
no firewall policy <seq>configRemove policy by sequence number
no firewall policy allconfigRemove all policies
show firewall policyEXECShow firewall policy table

seq: sequence number (evaluated lowest-first)   src/dst: IP address + wildcard mask, or any   proto: ip (all), icmp, tcp, udp   port: optional (tcp/udp only)   Implicit deny all at end (stateless — return traffic must also be permitted)
Example 1: firewall policy 10 permit 192.168.1.0 0.0.0.255 any any icmp  →  Allow ICMP from 192.168.1.0/24 to anywhere
Example 2: firewall policy 20 permit any any 10.0.0.1 0.0.0.0 tcp 443  →  Allow HTTPS to host 10.0.0.1
Example 3: firewall policy 999 deny any any any any ip  →  Explicit deny all (same as implicit default)

Tunnel interfaces create virtual point-to-point links between routers over the existing underlay network. Packets routed to a tunnel subnet are encapsulated and forwarded through the underlay to the peer.
Example (R1 side):
crypto isakmp policy 10 → encryption aes / hash sha / authentication pre-share / group 14
crypto ipsec transform-set MYSET esp-aes esp-sha-hmac
interface tunnel 0 → ip address 10.0.0.1 255.255.255.252 → tunnel source GigabitEthernet0/0 → tunnel destination 203.0.113.2
ip route 192.168.2.0 255.255.255.0 10.0.0.2  →  Route remote subnet via tunnel

Tips

Click device tabs or diagram icons to switch devices
Tab = auto-complete, ↑↓ = command history
VLAN colors appear on links in the network diagram
Routers need ip route to forward packets between networks
PCs need ip default-gateway to reach other networks
NAT requires ip nat inside/outside on interfaces + translation rules
Firewalls have an implicit deny all — add firewall policy to permit traffic
ACLs need ip access-group <num> in|out on an interface to take effect
Extended ACLs (100-199) filter by source, destination, protocol, and port
Switches become L3 when SVIs are configured: interface vlan 10
LACP bond: bond-group Bond0 on paired interfaces for failover
VPN tunnel: interface tunnel 0 + tunnel source/destination to create virtual links
Route remote subnets via tunnel IP: ip route 192.168.2.0 255.255.255.0 10.0.0.2
